问题1

问题描述

在数据库中创建名为user的table时会报错，因为SQL Server将user当成了关键字而非表名。

create table user
(
    userId int,
    username varchar(10),
    password varchar(10),
    email varchar(25),
    phone varchar(11),
    address varchar(50),
    primary key(userId)
);

解决方法

将user替换为[user]，这样SQL Server就不会将user认为是关键字了。

类似的关键字有order。

问题2

问题描述

在使用Hibernate的过程中,执行如下语句

String sql="from User u where u.username=? and u.password=?";

会报错，因为Hibernate在将上述语句转化为SQL语句过程中并不会帮你将user替换为[user]

解决方法

打开User.hbm.xml

定位到如下语句

<hibernate-mapping>
  <class name="vo.User" table="user" schema="dbo" catalog="ofos">...</class>
</hibernate-mapping>
               

将'user'替换为'[user]'即可。
